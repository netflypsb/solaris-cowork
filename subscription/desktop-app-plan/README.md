# Desktop App Development Plan: Clerk Auth + OpenRouter Key Retrieval

## Context
The desktop app (Electron/Tauri) needs to:
1. Authenticate users via the Solaris website (Clerk handles auth on the web)
2. Retrieve the user's subscription status and OpenRouter API key
3. Use the key for AI features

Users do NOT create API keys manually. Keys are created automatically when they subscribe on the website via Stripe.

---

## Prerequisites
- User has signed up on https://solaris-ai.xyz/
- User has subscribed (Stripe payment completed)
- OpenRouter API key was auto-generated by the website's Stripe webhook
- User has downloaded and installed the desktop app

---

## Architecture

```
Desktop App
  ├── Step 1: Authentication (one-time, opens system browser)
  │     └── Opens https://solaris-ai.xyz/auth/desktop
  │           └── User signs in via Clerk on the website
  │           └── Website generates HMAC token (5-min expiry)
  │           └── Deep link: solaris://auth/callback?token=<custom-token>
  │
  ├── Step 2: Token Verification + Key Retrieval (single API call)
  │     └── POST https://solaris-ai.xyz/api/auth/verify-desktop-token
  │           Body: { "token": "<custom-token>" }
  │           Returns: {
  │             valid: true,
  │             userId: "user_...",
  │             email: "user@example.com",
  │             hasSubscription: true,
  │             subscription: { status, tier, currentPeriodEnd, cancelAtPeriodEnd },
  │             apiKey: { key: "sk-or-v1-...", label, name, creditLimit }
  │           }
  │
  ├── Step 3: Store credentials locally
  │     └── Store userId, email, and OpenRouter API key in OS keychain (keytar)
  │     └── The 5-min token is NOT stored — it's single-use
  │
  └── Step 4: OpenRouter AI Calls (direct, no website involved)
        └── Uses stored apiKey.key directly with OpenRouter API
        └── POST https://openrouter.ai/api/v1/chat/completions
```

### Key Design Decisions
- **One-time auth**: Desktop app only contacts the website during sign-in. After that, it uses the OpenRouter key directly.
- **No ongoing website dependency**: AI calls go directly to OpenRouter, not through the website.
- **Token is single-use**: The 5-min HMAC token is used once to verify identity and retrieve the key, then discarded.
- **Re-auth triggers**: User signs out, subscription changes, key rotation, or credit limit reached.

---

## Website API Endpoints Available for Desktop App

These endpoints are **already implemented** on the website:

| Endpoint | Method | Auth | Purpose |
|----------|--------|------|---------|
| `/auth/desktop` | PAGE | Clerk (web) | Shows sign-in UI, generates HMAC token, shows deep link button |
| `/api/auth/verify-desktop-token` | POST | HMAC token | Verifies token, returns userId, email, subscription, **full OpenRouter key** |

**Important**: The `/api/user/subscription` endpoint uses Clerk session cookies (browser-only). The desktop app should **NOT** use it. Use `/api/auth/verify-desktop-token` instead — it returns everything the desktop app needs.

---

## Website Auth Page Behavior (`/auth/desktop`)

This page is a server component that:
1. If user is **not signed in** → renders an embedded Clerk `<SignIn />` component with `forceRedirectUrl="/auth/desktop"`
2. If user **is signed in** → generates an HMAC-signed token (5-min expiry), stores token hash in `desktop_sessions` table, and displays a button linking to `solaris://auth/callback?token=<token>`

The token format is: `base64url(JSON payload).base64url(HMAC-SHA256 signature)`
- Payload: `{ sub: userId, email, iat, exp }`
- Signed with: `DESKTOP_AUTH_SECRET` env var (falls back to `CLERK_SECRET_KEY`)

### STEP 2: Register Custom Protocol Handler (`solaris://`)
File: `src/main/index.ts`

```typescript
import { app, BrowserWindow } from 'electron';
import path from 'path';

const authManager = new AuthManager();

// Register solaris:// protocol BEFORE app is ready
if (process.defaultApp) {
  if (process.argv.length >= 2) {
    app.setAsDefaultProtocolClient('solaris', process.execPath, [
      path.resolve(process.argv[1])
    ]);
  }
} else {
  app.setAsDefaultProtocolClient('solaris');
}

// Handle protocol on macOS
app.on('open-url', (event, url) => {
  event.preventDefault();
  handleProtocolUrl(url);
});

// Handle protocol on Windows (single instance lock)
const gotTheLock = app.requestSingleInstanceLock();
if (!gotTheLock) {
  app.quit();
} else {
  app.on('second-instance', (event, commandLine) => {
    const url = commandLine.find(arg => arg.startsWith('solaris://'));
    if (url) handleProtocolUrl(url);

    // Focus the main window
    const mainWindow = BrowserWindow.getAllWindows()[0];
    if (mainWindow) {
      if (mainWindow.isMinimized()) mainWindow.restore();
      mainWindow.focus();
    }
  });
}

function handleProtocolUrl(url: string) {
  const parsed = new URL(url);
  if (parsed.hostname === 'auth' && parsed.pathname === '/callback') {
    const token = parsed.searchParams.get('token');
    if (token) {
      authManager.handleCallback(token);
    }
  }
}
```

### STEP 3: Implement Auth Manager
File: `src/main/auth.ts`

```typescript
import { shell } from 'electron';
import * as keytar from 'keytar';

const SERVICE_NAME = 'solaris-cowork';
const APP_URL = 'https://solaris-ai.xyz';

interface AuthData {
  userId: string;
  email: string;
  hasSubscription: boolean;
  apiKey: string | null;
}

class AuthManager {
  private mainWindow: BrowserWindow | null = null;

  setMainWindow(win: BrowserWindow) {
    this.mainWindow = win;
  }

  // Sign in: open system browser to website auth page
  async signIn(): Promise<void> {
    await shell.openExternal(`${APP_URL}/auth/desktop`);
  }

  // Handle solaris://auth/callback?token=xxx
  // This is called by the protocol handler when the browser redirects back
  async handleCallback(token: string): Promise<boolean> {
    try {
      // Verify token with website — this single call returns everything we need
      const res = await fetch(`${APP_URL}/api/auth/verify-desktop-token`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ token })
      });

      if (!res.ok) {
        console.error('Token verification failed:', res.status);
        this.mainWindow?.webContents.send('auth:error', 'Authentication failed');
        return false;
      }

      const data = await res.json();

      // Store user info in OS keychain (secure storage)
      await keytar.setPassword(SERVICE_NAME, 'user_id', data.userId);
      await keytar.setPassword(SERVICE_NAME, 'email', data.email);

      // Store the OpenRouter API key if subscription is active
      if (data.apiKey?.key) {
        await keytar.setPassword(SERVICE_NAME, 'openrouter_key', data.apiKey.key);
      }

      // Notify renderer that auth succeeded
      this.mainWindow?.webContents.send('auth:success', {
        userId: data.userId,
        email: data.email,
        hasSubscription: data.hasSubscription,
        hasApiKey: !!data.apiKey?.key,
      });

      return true;
    } catch (err) {
      console.error('Auth callback error:', err);
      this.mainWindow?.webContents.send('auth:error', 'Authentication failed');
      return false;
    }
  }

  // Check if user has stored credentials (app startup check)
  async isAuthenticated(): Promise<boolean> {
    const userId = await keytar.getPassword(SERVICE_NAME, 'user_id');
    return !!userId;
  }

  // Get stored user data
  async getUser(): Promise<AuthData | null> {
    const userId = await keytar.getPassword(SERVICE_NAME, 'user_id');
    if (!userId) return null;

    const email = await keytar.getPassword(SERVICE_NAME, 'email') || '';
    const apiKey = await keytar.getPassword(SERVICE_NAME, 'openrouter_key');

    return {
      userId,
      email,
      hasSubscription: !!apiKey,
      apiKey,
    };
  }

  // Get stored OpenRouter key
  async getOpenRouterKey(): Promise<string | null> {
    return keytar.getPassword(SERVICE_NAME, 'openrouter_key');
  }

  // Sign out: clear all stored credentials
  async signOut(): Promise<void> {
    await keytar.deletePassword(SERVICE_NAME, 'user_id');
    await keytar.deletePassword(SERVICE_NAME, 'email');
    await keytar.deletePassword(SERVICE_NAME, 'openrouter_key');
    this.mainWindow?.webContents.send('auth:signed-out');
  }
}

export default AuthManager;
```

### STEP 4: Implement AI Service
File: `src/main/ai-service.ts`

```typescript
import AuthManager from './auth';

class AIService {
  private authManager: AuthManager;
  private cachedKey: string | null = null;

  constructor(authManager: AuthManager) {
    this.authManager = authManager;
  }

  // Get the OpenRouter API key from OS keychain
  async getApiKey(): Promise<string> {
    if (this.cachedKey) return this.cachedKey;

    const key = await this.authManager.getOpenRouterKey();
    if (!key) {
      throw new Error('No API key available. Please subscribe at https://solaris-ai.xyz/pricing');
    }

    this.cachedKey = key;
    return key;
  }

  // Make an AI chat completion request directly to OpenRouter
  async chat(messages: any[], model: string = 'anthropic/claude-sonnet-4'): Promise<any> {
    const apiKey = await this.getApiKey();

    const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json',
        'HTTP-Referer': 'https://solaris-ai.xyz',
        'X-Title': 'Solaris Cowork Desktop'
      },
      body: JSON.stringify({ model, messages })
    });

    if (!response.ok) {
      if (response.status === 402) {
        this.cachedKey = null;
        throw new Error('Credit limit reached. Please check your subscription at https://solaris-ai.xyz/dashboard/api-key');
      }
      if (response.status === 401) {
        this.cachedKey = null;
        throw new Error('API key invalid. Please sign in again.');
      }
      throw new Error(`AI request failed: ${response.statusText}`);
    }

    return response.json();
  }

  // Clear cached key (call on sign out or re-auth)
  clearCache() {
    this.cachedKey = null;
  }
}

export default AIService;
```

import { contextBridge, ipcRenderer } from 'electron';

contextBridge.exposeInMainWorld('solaris', {
  auth: {
    signIn: () => ipcRenderer.invoke('auth:signIn'),
    signOut: () => ipcRenderer.invoke('auth:signOut'),
    isAuthenticated: () => ipcRenderer.invoke('auth:isAuthenticated'),
    getUser: () => ipcRenderer.invoke('auth:getUser'),
    onSuccess: (cb: (data: any) => void) => {
      ipcRenderer.on('auth:success', (_, data) => cb(data));
    },
    onError: (cb: (msg: string) => void) => {
      ipcRenderer.on('auth:error', (_, msg) => cb(msg));
    },
    onSignedOut: (cb: () => void) => {
      ipcRenderer.on('auth:signed-out', () => cb());
    },
  },
  ai: {
    chat: (messages: any[], model?: string) =>
      ipcRenderer.invoke('ai:chat', messages, model),
  }
});

import { ipcMain } from 'electron';
import AuthManager from './auth';
import AIService from './ai-service';

export function registerIpcHandlers(authManager: AuthManager, aiService: AIService) {
  ipcMain.handle('auth:signIn', () => authManager.signIn());
  ipcMain.handle('auth:signOut', async () => {
    aiService.clearCache();
    await authManager.signOut();

  ipcMain.handle('auth:getUser', () => authManager.getUser());

}

## Re-Authentication Flow

The desktop app needs to re-authenticate (open browser again) when:
1. **User signs out** — clears keychain, shows sign-in screen
2. **API key returns 401/402** — key was revoked or credit limit hit
3. **User clicks "Refresh Subscription"** — re-opens browser auth to fetch updated key
4. **Subscription expired** — prompt user to re-subscribe, then re-auth

## Security Considerations

1. **Token is single-use**: The 5-min HMAC token is only used once to verify identity and retrieve the API key. It is NOT stored.
2. **API key storage**: Use `keytar` (OS keychain) to store the OpenRouter key. Never use localStorage, plain files, or electron-store for secrets.
3. **Protocol handler validation**: Only process `solaris://auth/callback` URLs. Ignore all other protocol URLs.
4. **HTTPS only**: All API calls to the website and OpenRouter must use HTTPS.
5. **Memory-only key cache**: Cache the API key in memory for performance, clear on sign out.
6. **Single instance**: Use `app.requestSingleInstanceLock()` to prevent multiple app instances.
7. **No Clerk dependency**: The desktop app does NOT use Clerk directly. All auth goes through the website.

## User Flow Summary

```
FIRST-TIME SIGN IN:
1. User opens desktop app → sees "Sign In" screen
2. Clicks "Sign In" → system browser opens https://solaris-ai.xyz/auth/desktop
3. User signs in with Google/email (Clerk handles this on the web)
4. Website generates 5-min HMAC token → shows "Open Solaris Desktop" button
5. Button links to solaris://auth/callback?token=xxx
6. Desktop app receives token via protocol handler
7. App calls POST /api/auth/verify-desktop-token with the token
8. Website returns: userId, email, subscription status, full OpenRouter API key
9. App stores userId, email, and API key in OS keychain
10. App shows main interface → user can now use AI features

SUBSEQUENT LAUNCHES:
1. App checks keychain for stored credentials
2. If found → show main interface immediately (no network call needed)
3. AI calls go directly to OpenRouter using stored key

RE-AUTH (when needed):
1. OpenRouter returns 401/402 → app prompts "Sign in again"
2. User clicks → browser opens → same flow as first-time
3. Updated key is stored, replacing the old one
```

## Stripe Product Configuration

No special Stripe product settings are needed. The website's Stripe webhook automatically:
- Creates an OpenRouter API key when a user subscribes
- Stores the full key in Supabase for desktop app retrieval
- Disables the key when subscription is canceled or expires
- Re-enables the key when subscription is renewed

The webhook handles these Stripe events:
- `checkout.session.completed` → provision key
- `customer.subscription.updated` → enable/disable key based on status
- `customer.subscription.deleted` → disable key
- `invoice.payment_failed` → mark as past_due
