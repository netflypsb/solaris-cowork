# Desktop App Development Plan: Clerk Auth + OpenRouter Key Retrieval

## Context
The desktop app (Electron/Tauri) needs to:
1. Authenticate users via Clerk (same account used on website)
2. Check subscription status via the website's API
3. Retrieve the user's OpenRouter API key (auto-generated on subscription)
4. Use the key for AI features

Users do NOT create API keys manually. Keys are created automatically when they subscribe on the website.

---

## Prerequisites
- User has signed up on https://solaris-ai.xyz/
- User has subscribed (Stripe payment completed)
- OpenRouter API key was auto-generated by the website webhook
- User has downloaded and installed the desktop app

---

## Architecture

```
Desktop App
  ├── Clerk Auth (Electron/Tauri browser window or deep link)
  │     └── Opens https://solaris-ai.xyz/sign-in?redirect=solaris://auth/callback
  │           └── Clerk session token returned
  ├── Subscription Check
  │     └── GET https://solaris-ai.xyz/api/user/subscription
  │           Headers: { Authorization: Bearer <clerk-session-token> }
  │           Returns: { hasSubscription, subscription, apiKey }
  └── OpenRouter AI Calls
        └── Uses apiKey.label (or fetched full key) for AI requests
```

### STEP 2: Install Clerk for Electron
Clerk doesn't have a native Electron SDK, so use one of these approaches:

**Option A: Deep Link Authentication (Recommended)**
1. Register custom protocol `solaris://` in Electron
2. Open system browser to `https://solaris-ai.xyz/auth/desktop`
3. Website handles Clerk sign-in, then redirects to `solaris://auth/callback?token=<session-token>`
4. Electron receives the token via protocol handler

**Option B: Embedded BrowserWindow**
1. Open a BrowserWindow pointing to `https://solaris-ai.xyz/sign-in`
2. Listen for navigation to callback URL
3. Extract session token from URL

**Option C: Clerk's `@clerk/clerk-js` package**
1. Install `@clerk/clerk-js` directly
2. Use Clerk's headless mode for token management
3. Open sign-in in system browser, handle callback

### STEP 4: Implement Electron Auth Module
File: `src/main/auth.ts`

```typescript
import { app, shell, protocol } from 'electron';
import * as keytar from 'keytar'; // Secure credential storage

const SERVICE_NAME = 'solaris-cowork';
const APP_URL = 'https://solaris-ai.xyz';

class AuthManager {
  // Sign in: open browser, handle callback
  async signIn(): Promise<void> {
    const authUrl = `${APP_URL}/auth/desktop`;
    await shell.openExternal(authUrl);
    // Protocol handler (registered in STEP 5) receives the callback
  }

  // Handle the solaris://auth/callback?token=xxx response
  async handleCallback(token: string): Promise<boolean> {
    // Verify token with website API
    const res = await fetch(`${APP_URL}/api/auth/verify-desktop-token`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ token })
    });

    if (!res.ok) return false;

    const data = await res.json();

    // Store credentials securely in OS keychain
    await keytar.setPassword(SERVICE_NAME, 'session_token', token);
    await keytar.setPassword(SERVICE_NAME, 'user_id', data.userId);
    await keytar.setPassword(SERVICE_NAME, 'email', data.email);

    return true;
  }

  // Check if user is authenticated
  async isAuthenticated(): Promise<boolean> {
    const token = await keytar.getPassword(SERVICE_NAME, 'session_token');
    if (!token) return false;

    // Verify token is still valid
    const res = await fetch(`${APP_URL}/api/auth/verify-desktop-token`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ token })
    });

    return res.ok;
  }

  // Sign out
  async signOut(): Promise<void> {
    await keytar.deletePassword(SERVICE_NAME, 'session_token');
    await keytar.deletePassword(SERVICE_NAME, 'user_id');
    await keytar.deletePassword(SERVICE_NAME, 'email');
  }

  // Get stored token
  async getToken(): Promise<string | null> {
    return keytar.getPassword(SERVICE_NAME, 'session_token');
  }
}
```

### STEP 5: Register Custom Protocol Handler
File: `src/main/index.ts`

```typescript
import { app, BrowserWindow } from 'electron';

// Register solaris:// protocol
if (process.defaultApp) {
  if (process.argv.length >= 2) {
    app.setAsDefaultProtocolClient('solaris', process.execPath, [
      path.resolve(process.argv[1])
    ]);
  }
} else {
  app.setAsDefaultProtocolClient('solaris');
}

// Handle protocol on macOS
app.on('open-url', (event, url) => {
  event.preventDefault();
  handleProtocolUrl(url);
});

// Handle protocol on Windows (single instance lock)
const gotTheLock = app.requestSingleInstanceLock();
if (!gotTheLock) {
  app.quit();
} else {
  app.on('second-instance', (event, commandLine) => {
    const url = commandLine.find(arg => arg.startsWith('solaris://'));
    if (url) handleProtocolUrl(url);

    // Focus the main window
    const mainWindow = BrowserWindow.getAllWindows()[0];
    if (mainWindow) {
      if (mainWindow.isMinimized()) mainWindow.restore();
      mainWindow.focus();
    }
  });
}

function handleProtocolUrl(url: string) {
  const parsed = new URL(url);
  if (parsed.hostname === 'auth' && parsed.pathname === '/callback') {
    const token = parsed.searchParams.get('token');
    if (token) {
      authManager.handleCallback(token);
    }
  }
}
```

### STEP 6: Implement Subscription & Key Retrieval
File: `src/main/subscription.ts`

```typescript
const APP_URL = 'https://solaris-ai.xyz';

class SubscriptionManager {
  private authManager: AuthManager;

  constructor(authManager: AuthManager) {
    this.authManager = authManager;
  }

  // Get subscription status and API key from website
  async getSubscriptionAndKey(): Promise<{
    hasSubscription: boolean;
    subscription: any;
    apiKey: any;
  }> {
    const token = await this.authManager.getToken();
    if (!token) throw new Error('Not authenticated');

    const res = await fetch(`${APP_URL}/api/user/subscription`, {
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
      }
    });

    if (!res.ok) {
      if (res.status === 401) throw new Error('Session expired');
      throw new Error('Failed to check subscription');
    }

    return res.json();
  }

  // Get OpenRouter API key for AI calls
  async getOpenRouterKey(): Promise<string | null> {
    const data = await this.getSubscriptionAndKey();
    if (!data.hasSubscription || !data.apiKey?.hasKey) return null;

    // The website API returns the key label (preview), not the full key
    // For the desktop app, we need a way to get the full key
    // Option 1: Store full key in OS keychain on first retrieval
    // Option 2: Website API returns encrypted key for desktop
    // Option 3: Desktop app uses a separate endpoint that returns the full key
    return data.apiKey.fullKey || null;
  }
}
```

### STEP 7: Implement AI Service
File: `src/main/ai-service.ts`

```typescript
class AIService {
  private subscriptionManager: SubscriptionManager;
  private cachedKey: string | null = null;

  constructor(subscriptionManager: SubscriptionManager) {
    this.subscriptionManager = subscriptionManager;
  }

  async getApiKey(): Promise<string> {
    if (this.cachedKey) return this.cachedKey;

    const key = await this.subscriptionManager.getOpenRouterKey();
    if (!key) throw new Error('No API key available. Please subscribe at https://solaris-ai.xyz/pricing');

    this.cachedKey = key;
    return key;
  }

  async chat(messages: any[], model: string = 'anthropic/claude-sonnet-4'): Promise<any> {
    const apiKey = await this.getApiKey();

    const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json',
        'HTTP-Referer': 'https://solaris-ai.xyz',
        'X-Title': 'Solaris Cowork Desktop'
      },
      body: JSON.stringify({ model, messages })
    });

    if (!response.ok) {
      if (response.status === 402) {
        // Credit limit reached
        this.cachedKey = null;
        throw new Error('Credit limit reached. Please renew your subscription.');
      }
      throw new Error(`AI request failed: ${response.statusText}`);
    }

    return response.json();
  }

  // Clear cached key (e.g., on sign out or key rotation)
  clearCache() {
    this.cachedKey = null;
  }
}
```

## Security Considerations

1. **Token storage**: Use `keytar` (OS keychain) — never localStorage or plain files
2. **Protocol handler**: Validate the callback URL origin
3. **API key caching**: Cache in memory only, clear on sign out
4. **HTTPS only**: All API calls to website must use HTTPS
5. **Token expiry**: Desktop tokens should expire (e.g., 30 days), re-auth required
6. **Single instance**: Prevent multiple app instances to avoid auth conflicts

## User Flow Summary

```
1. User opens desktop app for the first time
2. App shows "Sign In" screen
3. User clicks "Sign In" → system browser opens to solaris-ai.xyz/auth/desktop
4. User signs in with Google/email (same as website account)
5. Website generates desktop token, redirects to solaris://auth/callback?token=xxx
6. Desktop app receives token, stores in OS keychain
7. App checks subscription → if active, retrieves OpenRouter key
8. User can now use AI features in the desktop app
9. On subsequent launches, app auto-checks stored token validity
```
